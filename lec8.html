[Lecture 8] Iterations

Why do we need logic in programs?

One sure thing that computers are often used for is to automate repetitive tasks. Repeating identical or similar tasks several times without making errors is something that computers do well and people do poorly. Humans get bored doing the same thing a million times – however for a computer, it is a pleasure (or rather the purpose)! The process of repeating a certain task several times is generally called (even colloquially..) iteration.

Some silly examples:
  1. Suppose you want to convince a friend to join a party and he is a total lazy ass! You ask him again and again, until he agrees. Now technically we say you iterate the task of requesting your friend until he agrees!
  2. Now, imagine you have 10 such lazy friends. In this case you'll have to iterate the process of convincing each of them 10 times!

Typically iterations are achieved in computer programs using constructs called 'loops'. Loops allow us to write concise, repeatable code that can be applied to each element in a collection or perform a task over and over again until some condition is met. When writing a loop, there are three essential components:

  1. An initialisation statement that specifies how the loop begins.
  2. A continuation (or termination) condition (a Boolean statement) that specifies whether the loop should continue to execute or terminate.
  3. An iteration statement that makes progress toward the termination condition. 

The initialisation statement is executed before the loop begins and serves as a way to set the loop up. Typically, the initialisation statement involves setting the initial value of some variable.

The continuation statement is a logical statement (that evaluates to true or false) that specifies if the loop should continue (if the value is true) or should terminate (if the value is false). Upon termination, code returns to a sequential control flow and the program.

The iteration statement is intended to update the state of a program to make progress toward the termination condition. If we didn’t make such progress, the loop would continue on forever as the termination condition would never be satisfied. This is known as an infinite loop , and results in a program that never terminates.

                                                         


As a simple example, consider the following outline.

• Initialise the value of a variable i to 1

• While the value of i is less than or equal to 10 . . . (continuation condition) • Perform some action (this is sometimes referred to as the loop body

• Iterate the variable i by adding one to its value

The code outline above specifies that some action is to be performed once for each value: i = 1, i = 2, . . . , i = 10, after which the loop terminates. Overall, the loop executes a total of 10 times. Prior to each of the 10 executions, the value of i is checked; as it is less than or equal to 10, the action is performed. At the end of each of the 10 iterations, the variable i is incremented by 1 and the termination condition is checked again, repeating the process. There are several different types of loops that vary in syntax and style but they all have the same three basic components.

Alright, how do we create loops in C?

Well, in C programming language there are two ways of achieving this:

    • the while-loop construct
    • the for-loop construct

In C programming language both are almost identical in terms of capabilities and there is almost never a case when one must be preferred over the other. Whereas in Python, for instance, they both work entire differently (we'll learn this later).

while loop - a simple example :

int i = 1; // i is the counter variable

while(i < 11){ // i < 11 is a boolean conditional

       printf("make pizza number %d\n", i); //Action that is iteratively performed so long as the conditional is True

       i = i + 1; //the value of i is updated every iteration

      }

for loop - a simple example :


The following for loop does exactly does what the while loop does. The only difference is that the initialization statment, the conditional and the incrementation are all nicely packed within the pair of parens after the for.

for(int i = 1; i < 11; i = i + 1){

     printf("make pizza number %d\n", i);

}

So long as you know why you are implementing iteration and what you want to do with it, there is almost never a case when you would prefer one over the other. 

And one more thing:

There is also this construct called the do-while construct which comes handy some times. 

int i = 0;

do{

     printf("make pizza number %d\n", i);

}

while(i < 10);




The subtle difference between the while construct and the do-while construct can best be appreciated by experimenting with them!

Infinite loops:

You may sometimes want to keep the loop running forever until it is broken intentionally. All the video games you play are actually infinite loops. They close only when when you click on "Quit Game" manually. Also, in fact most of the programs that you use on your computer (eg: Browser, VS Code, Spotify etc) also also infinite loops -- for none of them close by themselves unless you explicitly click on the close button. 

As such creating an infinite loop isn't hard. All you need to do is just make sure that the Boolean conditional is always true. Have a look at this example:

while(1 == 1){
       printf("some actions\n");
}

In here, because the conditional statement is always true (because 1 is always equal to itself!), the action will be performed iteratively until the end of the universe. If you incorporate 'quitting upon command' feature you can achieve that using the keyword 'break'. 

while(1 == 1){
       printf("some actions\n");
       int i;
       scanf("%d", &i);
       if(i == -1){
                      break;
        }

 }

What the above program does is that it prompts you to enter an integer at every iteration and unless the user enters -1 the loop continues. But when the user does enter -1, the loop breaks thanks to the 'break' command.
