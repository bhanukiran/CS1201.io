C vs Python?

24/01/2021

To think of different programming languages (C, C++, Python, Java, etc ) as mutual rivals is a serious misunderstanding of reality. Fact is that every programming language that you find today would have emerged in a specific historical context and was developed to solve a specific class of computational problems. Therefore, it is rather inappropriate to see any one of them as universally the best. Nonetheless, we can always say that a particular language is the most suitable for a specific kind of tasks - usually the tasks for which it was developed to begin with. For example: C was a language developed in 1970's with the purpose of building an operating system in mind; whereas Java was developed in 1990's - the time that coincides with rapid development of internet - to build applications that run across computers that are connected across a network. So naturally, what C can do easily - C does it easily; and likewise what Java can do it easily - Java does it easily - because its very purpose of existence is associated with it.

As it happens Python is a language that was also developed in 1990's, but with a different purpose in mind. When Guido van Rossum (a Dutch programmer) developed Python[1] he wanted to create a quick and easy way of building prototypes. Now, what does that mean? Suppose you have an idea to design a rocket that can enter Jupiter's orbit with very little fuel cost carrying much higher payload. For people to believe that your idea works, they expect you to come up with a prototype - a miniature version of the real rocket itself. If the protype fails to stand the tests, there's no way the actual rocket could suceed; on the contrary if the prototype succeeds there are high chances that the actual rocket may succeed. In the same spirit, suppose you conceive a killer mobile application that you think can replace, say WhatsApp, in terms of some specific features etc, you should develop a prototype - real quick - to convice the world that your idea is worth the attention. It is for this purpose that Python generally comes handy. It enables you write programs with non-trivial functionalities in a very short time - rapid prototype development, as they call it in the business. So naturally if Python feels simple, how is it a surprise? It is supposed to be simple for humans! But eventually when you want to deploy commercial applications that ought to run very fast across devices, you may want to port it to a lower level language such as C/C++. All popular web-browsers such as Firefox, Google Chrome etc and several famous videogames and applications such as Counterstrike, Call of duty, Adobe Photoshop etc are all built using C, C++ because 'speed' is a very crucial parameter. Therefore learning C is still very much relevant for every engineer despite the fact that the language has been around for more than 50 years.

In recent years, thanks to faster CPUs and cheaper avialbility of RAMs, Python has emerged as a favourite to many academicians as well as industry programmers because it provides a quick way of solving a broad class problems - ranging from scientific computing to web-development - without having to write long tedious programs in lower level languages. It has come to be particularly well known among developers and researchers of Machine learning which makes it a very important language to master. But, one should also keep in mind that most of the heavy duty tools that work in the background of machine learning programs are written in speed efficient languages such as C, C++. As a matter of fact, the language Python (the Python interpretor, that is the engine that makes Python work) itself is written in C! Imagine how silly one sounds to argue that bread is a rival to flour! Can bread ever compete to be better than the flour? The question itself makes little sense!

Conclusion

Python is a high level language (higher because it is closer to human logic than to machine logic) used to solve specific kinds of problems by developing quick solutions. C, on the other hand is a lower language and has been used in developing some very important applicaitons that will continue to be important in forseeable future. Therefore, as beginners, rather than seeing them as rivals one trying to dethrone the other, we see them as two different tools (both very important) each suited for specific tasks. It is all about choosing the right tool for the right job.
